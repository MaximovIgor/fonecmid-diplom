# Инструкция по начальной настройки
 
## Настройка пользователей

В разделе Администрирование -> Настройка пользователей и прав создать пользователей
Для работы программы необходимо создать пользователей:
- Администратор
- Специалист по обслуживанию клиентов
- Менеджер
- Бухгалтер
- КадровикРасчетчик

## Начтройка связи с Telegram

В разделе Администрирование -> Сервис указать Идентификатор группы для оповещения (Для теста: -1003163911586)
В разделе Администрирование -> Сервис указать Токен управления Телеграм-ботом (Для теста: 8397680616:AAF28CtESSdijXiOVRfP_ALRQMXUjuiBVTA)
В разделе Администрирование -> Обслуживание -> Регламентные операции -> Отправка уведомленией телеграм боту Регламентные и фоновые задания настроить рассписание выполнениея задания
В разделе Расчет управленческой зарплаты -> Максимальное число дней отпуска указать плановое количество дней отпуска

## Настройка условий работы специалистов

Для корректного проведения документа ОбслуживаниеКлиента необходимо предварительно установить для специалиста в разделе 
Расчет управленческой зарплаты -> Условия оплаты сотрудников оклад специалиста и % от работ


## Расчет заработной платы

Настроить графики работы
Расчет управленческой зарплаты -> Заполнить графики работы
Расчет управленческой зарплаты -> НДФЛ

# Подготовка базы для запуска сценария тестирования

Внести изменения в файл config.cfg. Добавить параметр DisableUnsafeActionProtection=.*. На проде такое делать не рекомендуется. Используется только для тестирования.
Создать базу менеджера тестирования. В прараметрах запуска -> Дополнительные паратметры запуска указать параметр  /TESTMANAGER
В рабочей базе в прараметрах запуска -> Дополнительные паратметры запуска указать параметр  /TESTCLIENT
